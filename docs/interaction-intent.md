# UI 设计意图（interaction-intent）

本文档说明“为什么每个交互这样做”，避免后续维护时把行为改坏。

## 1. 窗口与标题栏

### 设计
- 面板窗口不可拖动（`ImGuiWindowFlags_NoMove`）。
- 标题栏仍有独立 hit 区用于“按下态”判定。

### 意图
- 保持像素级构图稳定，不让用户把面板拖离预期位置。
- 保留按下视觉反馈，统一鼠标 pressed 态表现。

### 关键规则
- 拖住标题后鼠标离开标题区：立即进入“锁定松开”。
- 未抬鼠标就算回到标题区，也不恢复按下；必须重新点击。

参考：`retro_skill_panel.cpp:58-65`。

## 2. Tab 切换（被动/主动）

### 设计
- Tab 由不可见按钮接管输入，贴图仅负责视觉。
- 点击后重置滚动并取消技能拖拽。

### 意图
- 切页后内容语义应是“新列表起点”，不是继承上一页滚动。
- 防止跨 tab 残留拖拽状态造成错觉。

参考：`retro_skill_panel.cpp:106-107`。

## 3. 技能列表滚动

### 设计
- 滚轮与滚动条拖拽都按“离散行高”吸附（按项滚动）。

### 意图
- 保持像素风 UI 的稳定对齐。
- 避免半行截断导致阅读与点击区域错位。

参考：`retro_skill_panel.cpp:177`, `retro_skill_panel.cpp:358`。

## 4. 技能拖拽显示

### 设计
- 点击技能图标进入拖拽；再次点击可取消。
- 拖拽图标跟随鼠标并保持抓取偏移。

### 意图
- 保持“抓住图标”的视觉一致性，而不是图标中心瞬移到鼠标。
- 支持轻量切换拖拽态，不需要额外交互按钮。

参考：`retro_skill_panel.cpp:231-235`, `retro_skill_panel.cpp:449-454`。

## 5. “+” 加点按钮

### 设计
- 状态分 normal / hover / pressed / disabled。
- 只有 release 且仍在按钮内时才生效（release-click）。

### 意图
- 减少误触；按下后拖出按钮再松手不会加点。
- 与经典 UI 行为一致。

参考：`retro_skill_panel.cpp:286-305`。

## 6. init 按钮

### 设计
- 与 `+` 一样使用 release-click 触发。
- 触发后将当前 tab 技能等级清零。

### 意图
- 防止按下即重置造成误操作。
- 与 `+` 保持输入模型一致，降低学习成本。

参考：`retro_skill_panel.cpp:417-443`。

## 7. 鼠标贴图状态机

### 设计
- normal / pressed / drag / hover-loop。
- hover 首次进入时随机从 normal.1 或 normal.2 开始，然后按 0.5s 切换。

### 意图
- 增加复古光标的“活性”，避免每次悬停都同一帧起始的机械感。
- 维持可预测节奏，避免随机闪烁。

参考：`retro_skill_panel.cpp:465-469`, `retro_skill_app.cpp:56-64`, `retro_skill_app.cpp:71-79`。

## 8. 为什么很多参数是硬编码

### 设计
- 坐标、尺寸、偏移大量使用常量。

### 意图
- 这是像素对齐 UI，不是弹性布局。
- 将这些值抽象成通用约束会破坏原图对位与交互热区一致性。

因此：如果要改视觉，请按“素材 + 命中区域 + 交互语义”三者一起改，不要只改某一项。
